### 2) DB migration — [`20260606120000_add_org_expense_summary/migration.sql`](20260606120000_add_org_expense_summary/migration.sql)

- `org_expense_agg_level` ENUM (`ministry` / `department`)
- `org_balance_import_run` — 적재 메타 (연도, 파일명, 기간, 건수)
- `org_expense_summary_row` — 사역원·부서별 `total_expense` 등
- **CREATE만** — `org_balance`·`org_budget` ALTER 없음

### 3) Go — [`org_expense_summary.go`](../org_expense_summary.go)

- `expenseSummaryCollector`: ministry 적재 루프에서 **1-pass** `(segment3, segment1)` 집계
- `total_expense = debitamt + creditamt` (기존 UI SUM과 동일 개념)
- `runMinistryOrgBalanceImport`: 단일 트랜잭션  
  run INSERT → delete `org_balance` → insert `org_balance` → insert summary → commit
- `org_budget` JOIN으로 `budget_amount`·`remaining_budget` 스냅샷
- migration 미적용 시 live 적재 **명확한 에러**

### 4) [`main.go`](../main.go)

- `handleMinistryBalance`: collector 인자 추가, `insertBudgetBalance`로 **org_balance INSERT 동일**
- live ministry: `runMinistryOrgBalanceImport` 호출 (delete+insert 트랜잭션 통합)
- dry-run: `org_expense_summary planned_insert department=N ministry=M` 출력

## 운영·배포 (요약)

```bash
# 1) DB (서버)
psql "$DATABASE_URL" -v ON_ERROR_STOP=1 \
  -f 2026/20260606120000_add_org_expense_summary/migration.sql

# 2) 코드
cd /opt/kcpc/onfinance && git pull && go mod download

# 3) 적재 (summary 자동 생성)
./scripts/run_all_imports.sh --dry-run
./scripts/run_all_imports.sh
```

- **소비 UI/API**: 재배포·재시작 **불필요**
- 상세: 개발 방안 **§14**, 서버 가이드

### 적재 후 확인

```sql
SELECT id, fiscalyear, source_filename, detail_row_count, imported_at
FROM org_balance_import_run ORDER BY imported_at DESC LIMIT 1;

SELECT aggregation_level, COUNT(*), SUM(total_expense)
FROM org_expense_summary_row
WHERE run_id = (SELECT id FROM org_balance_import_run ORDER BY imported_at DESC LIMIT 1)
GROUP BY aggregation_level;
```
