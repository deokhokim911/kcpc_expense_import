## 변경 요약

### 1) `scripts/run_all_imports.sh`

- **미리보기(dry-run) → 사용자 확인 → 실제 적재** 흐름 고정
  - `--dry-run/-n`: 미리보기만 수행 후 종료
  - `--yes/-y` 또는 `RUN_ALL_IMPORTS_YES=1`: 확인 프롬프트 생략
- **env 로딩 안정화**
  - `run_all_imports.env`가 CRLF여도 동작하도록 `tr -d '\r'`로 정리한 임시 파일을 `source`
- **환경변수 우선순위 버그 수정**
  - 실행 전 셸에 `VAR=""` 형태로 **빈 값만 export** 되어 있던 경우, env 로드 후 값이 사라지던 케이스를 방지
  - 이제는 **비어 있지 않은 값**이 있을 때만 env 값을 덮어씀
- **DB 연결 정보 사전 체크**
  - 미리보기/적재 모두 DB 조회가 필요하므로 `DATABASE_URL` 또는 `PGHOST/PGPORT/PGUSER/PGPASSWORD/PGDATABASE` 미설정 시 즉시 안내 후 종료

### 2) `main.go`

- `DATABASE_URL`에 포함될 수 있는 `schema=...` 쿼리 파라미터 제거
  - 일부 도구(예: Prisma)가 붙이는 `?schema=public`은 `lib/pq` 경로에서 서버 설정으로 해석되어
    `pq: unrecognized configuration parameter "schema"` 오류를 유발
  - 본 프로젝트는 쿼리에서 `public.`를 명시하므로 `schema` 파라미터는 필요하지 않음
- 사역원 dry-run에서 **DB 삭제 예정 건수 + 파일명 + 적재 예정 건수**를 출력하도록 개선

### 3) `fund_activity_fiscal_replace.go`

- fund activity fiscal-year replace dry-run 출력에
  - 삭제 예정(import_run/summary_row/detail_line)
  - 파일명(summary/detail)
  - 적재 예정(import_run=1, summary_row/detail_line)
  를 명시적으로 추가

### 4) `scripts/run_all_imports.env_sample`

- CRLF 안내 + DB 환경변수 안내 + `RUN_ALL_IMPORTS_YES=1` 안내 추가
